Decision-making

I structured navigation by language first, then version, to match how users think—choose language, then API version—while keeping page structure consistent.

User experience improvements

It makes navigation clearer, with easy access to all versions and languages, and consistent structure helps users switch between V1 and V2 smoothly.

Trade-offs

There’s some duplication and slightly deeper navigation, but it improves clarity and reduces user confusion.

Scalability

It scales well—new languages or versions can be added without changing the existing structure.
